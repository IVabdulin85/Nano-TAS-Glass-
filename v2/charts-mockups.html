02 / Чарт-модернизация · ARGUS RESEARCH

Витрина альтернатив для 4 диаграмм v2

На каждый из 4 существующих графиков — по 3 современных альтернативы с живыми визуализациями на реальных данных проекта. Каждый вариант — со своим takeaway-заголовком (как делают FT, Bloomberg, McKinsey). Выбери лучший в каждой категории.

→ P&L 5 лет → Кумулятивный FCF → Структура OPEX → Чувствительность → Решение
Слот 01 / P&L 5 лет

Выручка · EBITDA · Чистая прибыль по годам

Показать динамику ключевых финансовых показателей за горизонт расчёта. Цель — «растёт ли проект, как меняется маржа, есть ли убыточные годы».

сейчас: Chart.js grouped bar
Вариант 01 · Tufte / FT-style
Small Multiples
Chart.js × 3 mini-bars · low effort
«Каждая метрика растёт двузначными темпами Y1→Y5 (CAGR ≥ 7%)»
Выручка
$36.2M
+7.0% CAGR
EBITDA
$22.7M
+12.6% CAGR
Net Income
$17.1M
+13.7% CAGR
Когда подходит: чтобы инвестор сразу увидел три отдельных «двигателя» проекта, не сравнивая высоты столбцов на одной шкале.
Плюс: каждая метрика видна без помех + явный CAGR. Минус: сложно сопоставить «маржу» визуально.
Вариант 02 · IB-стандарт
Waterfall (Income Statement)
ECharts · Goldman Sachs / McKinsey style
«Из $27.6M выручки $10.3M превращаются в чистую прибыль (37% conversion)»
Когда подходит: когда нужно показать «как из выручки получается прибыль» — каждый отрицательный шаг подсвечивает статью расхода.
Плюс: любим IB / PE / стратегов. Минус: показывает только один год (можно сделать toggle Y1/Y3/Y5).
Вариант 03 · McKinsey combo
Bar + Margin Line + Annotations
Chart.js + annotation plugin
«Маржа EBITDA удерживается ≥51%, NI растёт быстрее выручки»
Когда подходит: хочется сохранить grouped bar (привычный формат), но дать дополнительный insight через линию маржи + reference target.
Плюс: двойной insight на одном чарте. Минус: слегка перегружено для первого взгляда.
Слот 02 / Кумулятивный диск. FCF

Накопленный денежный поток с учётом дисконта

Когда проект окупится — момент перехода кумулятивного диск. FCF из минуса в плюс. Сейчас линия с заливкой + горизонталь break-even.

сейчас: Chart.js line + break-even ref
Вариант 01 · FT / NYT-style
Conditional Area (red→green)
ECharts · visualMap by Y-value
«Break-even в Году 4 — переход из красной зоны в зелёную»
Когда подходит: визуально-эмоционально показать переход проекта из убытка в прибыль — глаз ловит цвет раньше, чем читает оси.
Плюс: моментальное «aha». Минус: нужна аккуратная палитра, чтобы цвета не были «крикливыми».
Вариант 02 · DCF-стандарт
Waterfall (CAPEX → FCF → Cum)
ECharts · IB pitch-deck стандарт
«−$35M CAPEX окупаются за 4 года, итог NPV +$15.6M»
Когда подходит: когда нужно показать аддитивность — большая отрицательная инвестиция, затем поэтапное возмещение, итоговый NPV.
Плюс: любимец IB/PE. Минус: теряется «время до break-even» (не видно конкретного года перехода).
Вариант 03 · Annotated journey
Line + Annotated Milestones
Chart.js + annotation plugin
«Дисконт. окупаемость 3.54 года · NPV $15.6M · IRR 29.3%»
Когда подходит: текущий формат, но с явными аннотациями ключевых точек (peak burn, payback, NPV).
Плюс: минимум изменений к v2, максимум storytelling. Минус: та же базовая визуализация, что сейчас.
Слот 03 / Структура OPEX

Разбивка операционных расходов

9 категорий (Сырьё 44.5% · Персонал · Обслуживание · Логистика · Газ · Администрация · Прочее · Электро · Вода). Donut для 8+ категорий — устаревший паттерн (Tufte/Knaflic).

сейчас: Chart.js doughnut · 8 cats
Вариант 01 · Modern part-of-whole
Treemap
ECharts · best для >5 категорий
«Сырьё — 44.5% всего OPEX, главный драйвер себестоимости»
Когда подходит: много категорий, нужно сравнить доли. Площади воспринимаются точнее углов pie.
Плюс: точный по восприятию + красивый. Минус: мелкие категории трудно подписать.
Вариант 02 · Maximum clarity
Horizontal Bar Ranked
Chart.js horizontal bar · sorted desc
«Топ-3 (сырьё + персонал + обслуживание) = 70% OPEX»
Когда подходит: самое читаемое и сравнимое решение для ранжированного списка категорий.
Плюс: сразу видны топ-драйверы. Минус: менее «эффектно» визуально.
Вариант 03 · Composition view
100% Stacked Bar
Chart.js stacked bar · single row
«Variable cost (сырьё+газ+электро+лог) = 64% структуры»
Когда подходит: компактно, можно сравнивать несколько лет/сценариев рядом (если будут).
Плюс: минимум места, легко добавить comparison. Минус: мелкие категории сжимаются.
Слот 04 / Чувствительность

Как меняются NPV / IRR при разных параметрах

Сейчас HTML-таблица с цветовыми ячейками. Альтернативы: настоящий heatmap, tornado (что влияет сильнее всего), 2D-карта (price × WACC).

сейчас: HTML table + color cells
Вариант 01 · Modern table replacement
1D Heatmap (Price × Metric)
ECharts heatmap + visualMap
«NPV вырастает в 5× при росте цены до $100/м²»
Когда подходит: текущая таблица, но как полноценный chart — с цветовой легендой, hover-tooltip, без табличных границ.
Плюс: привычная структура. Минус: остаётся одномерным (только цена).
Вариант 02 · IC / Monte-Carlo стандарт
Tornado Chart
Chart.js horizontal divergent bar
«Цена — главный драйвер NPV, вдвое сильнее CAPEX и OPEX»
Когда подходит: ранжирование драйверов по влиянию на NPV (±20% по каждому параметру). Любимый формат investment committees.
Плюс: сразу видно, на что обратить внимание. Минус: нужно объяснить методику.
Вариант 03 · PE-стандарт
2D Heatmap (Price × WACC)
ECharts heatmap · 6×7 grid
«Зона положительного NPV — при цене ≥$60 и WACC ≤16%»
Когда подходит: двумерная sensitivity matrix — самое сильное в PE-моделях. Видна вся «зона выживания» проекта.
Плюс: максимум информации. Минус: требует понимания heatmap-чтения.
